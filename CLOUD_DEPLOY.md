# 云端部署指南（腾讯云 SCF · 免费）

把单词机器人部署到腾讯云 SCF（Serverless）后，机器人 7×24 在线，不再依赖你的电脑开关。
公众号服务器配置照常启用（机器人照常干活），关注欢迎语、查词、「我学了啥」、「我要代码」全部永远可用。

> 适合：想"免运维、永远在线、零服务器费用"的个人号。
> 前提：需要一个**腾讯云账号**并完成**实名认证（身份证）**。SCF 有免费额度，个人低频使用基本不花钱。

---

## 一、准备腾讯云

1. 注册并登录 [腾讯云](https://cloud.tencent.com/)（用微信扫码即可）。
2. 完成**实名认证**（右上角头像 → 认证）：个人认证选"个人"，填身份证，几分钟审核过。
3. 开通**云函数 SCF** 服务（首次进入会提示开通，免费）。
4. （可选，非必须）如需让"词库"跨实例永久保存，可开通**对象存储 COS**（见文末"进阶"）。

## 二、创建 Web 函数

1. 进入 [SCF 控制台](https://console.cloud.tencent.com/scf/list) → **新建** → 选择**Web 函数**。
2. 地域：选离你近的（如"广州"）。
3. 运行环境：**Python 3.9**（或 3.10，boot 脚本已兼容）。
4. 函数名称：`wechat-word-bot`。
5. 上传方式：**本地上传 zip 包** → 选择本项目打包好的 `WeChat-word-bot-cloud.zip`。
   - ⚠️ zip 里必须含 `scf_bootstrap` 和所有 `.py` 文件，平台会以它为准启动。
6. 高级配置 → 启动命令保持默认（用 zip 内的 `scf_bootstrap`）。
7. 点**完成**，等待创建。

## 三、配置环境变量（关键）

在函数「函数管理 → 函数配置 → 编辑」里添加以下环境变量（**不要写在代码里，避免泄露**）：

| 变量名 | 值 | 说明 |
|--------|----|------|
| `WECHAT_CHANNEL` | `official` | 走公众号通道 |
| `MP_TOKEN` | 你的 Token | 必须和公众号后台"服务器配置"里的 Token **完全一致** |
| `MP_APPID` | 你的 AppID | 公众号后台获取 |
| `MP_APPSECRET` | 你的 AppSecret | 公众号后台获取 |
| `MP_CALLBACK_PATH` | `/wechat` | 回调路径 |

> 其余变量（`PORT=9000`、`DATA_DIR=/tmp`）已由 `scf_bootstrap` 自动设置，不用管。

## 四、创建 API 网关触发（拿到公网 URL）

1. 函数「触发管理 → 创建触发器」→ 触发方式选 **API 网关（HTTP 触发）**。
2. 新建 API 服务（或选已有），前端类型 HTTP，勾选"启用集成响应"。
3. 创建后，触发器会给出一个**访问路径**，形如：
   `https://service-xxxx-xxx.ap-guangzhou.tencentcs.com/release/`
   记下这个地址（后面要拼上 `/wechat`）。

## 五、回填公众号"服务器配置"

1. 公众号后台 → 设置与开发 → 基本配置 → **服务器配置** → 修改。
2. URL 填：`https://<上面的网关地址>/wechat`
   （例如 `https://service-xxxx.ap-guangzhou.tencentcs.com/release/wechat`）
3. Token 填：和 `MP_TOKEN` 环境变量**完全相同**的值。
4. 消息加解密方式：**明文模式**。
5. 点「提交」→ 显示"配置成功"即通。

> 此时旧的 natapp 隧道可以关掉了，机器人已在云端运行。

## 六、验证

- 给公众号发一个英文单词（如 `apple`）→ 应收到释义（说明在线且查词通）。
- 发「我学了啥」→ 应收到统计。
- 发「我要代码」→ 应收到源码下载地址。
- 取消关注再关注 → 应收到新欢迎语（机器人云端常驻，必达）。

---

## 七、国内访问词典 API 的风险提示（重要）

机器人查词调用的是免费公开 API：
- `api.dictionaryapi.dev`（英文释义）—— **服务器在境外**，国内（腾讯云）可能延迟高或偶发连不上。
- `api.mymemory.translated.net`（翻译）—— 国内通常可达。
- `translate.googleapis.com`（翻译兜底）—— 国内多被墙。

**后果**：在云端，英文单词查词可能偶尔失败（返回"未找到/网络异常"），但翻译类功能通常正常。

**如果查词经常失败**，有两个解法（告诉我，我帮你改）：
1. 把词典源换成国内可达的（如有道/金山词霸免费接口，部分需 key）。
2. 给机器人加一层缓存/重试，或改用国内镜像。

## 八、进阶：让"词库"永久保存（可选）

SCF 的 `/tmp` 在每个**冷启动**的新实例上会清空，所以：
- 机器人**在线期间**（同一实例）词库正常累积；
- 长时间无请求后实例回收，下次冷启动词库可能"归零"，导致「我学了啥」数字偏小。

要彻底持久化，把 `word_history.json` 存到 **COS 对象存储**（开通 COS → 建桶 → 在环境变量加 `COS_BUCKET`/`COS_REGION`/`SECRETID`/`SECRETKEY`，我可帮你把读写改到 COS）。个人低频使用一般影响不大，可先不管。

---

## 九、费用

SCF 按调用量+运行时长计费，免费额度内（每月一定量）**不花钱**。个人公众号低频使用通常长期免费。超出部分才计费，届时控制台会有提醒。

## 十、回滚到本地

不想用云了？在公众号后台"服务器配置"把 URL 改回你的 natapp 地址 + 本地 `python3 main.py` 即可，代码无需改动。
